# OctoAcme — Role Alignment & Handoff Checklist

## Purpose
Provide a lightweight checklist to confirm cross-functional role ownership and handoffs across the delivery lifecycle.

## How to use
- Complete during planning and revisit at each milestone/release.
- Mark N/A only when the role is truly not needed for the project scope.

## Role Ownership
- [ ] Product Manager assigned (vision, prioritization, success metrics)
- [ ] Project Manager assigned (timeline, dependencies, risk communication)
- [ ] Developer owner(s) assigned (implementation, code quality, test automation)
- [ ] UX/UI Designer assigned (user flow, interaction, accessibility expectations)
- [ ] Business Analyst assigned (requirements clarity, business rules, edge cases)
- [ ] QA/Testing owner assigned (test strategy, defect triage, release quality signal)
- [ ] DevOps owner assigned (CI/CD, environment readiness, monitoring, rollback readiness)
- [ ] Technical Writer/documentation owner assigned (release notes, user/internal docs)

## Key Handoffs by Phase
### Planning
- [ ] Requirements and acceptance criteria reviewed by PdM, BA, Dev, and QA
- [ ] UX/UI artifacts shared and acknowledged by Dev and QA
- [ ] DevOps dependencies and environment needs captured in backlog
- [ ] Documentation deliverables identified and linked to backlog items

### Execution
- [ ] PRs include acceptance criteria and cross-functional notes when applicable
- [ ] QA test progress and defect triage cadence agreed
- [ ] UX/UI and BA clarifications handled within sprint/iteration cadence
- [ ] DevOps readiness updates included in weekly delivery sync

### Release
- [ ] QA quality sign-off captured
- [ ] DevOps deployment and rollback checks complete
- [ ] Technical Writer (or owner) confirms release notes and documentation updates
- [ ] PM/PdM and stakeholders receive release communication summary
